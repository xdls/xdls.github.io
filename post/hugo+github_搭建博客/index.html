使用环境

操作系统：windows11

子系统：wsl2

发行版：openSUSE

安装hugo

安装Homebrew1

一开始用官方网站的安装方法，一直安装不上，后来用Google搜索如何安装，才安装成功。

rm Homebrew.sh ; wget https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh ; bash Homebrew.sh

安装hugo2

brew install hugo

生成站点

使用Hugo快速生成站点，比如希望生成到 /path/to/site 路径：

hugo new site /path/to/site

这样就在 /path/to/site 目录里生成了初始站点，进去目录：

cd /path/to/site

创建文章

创建一个 about 页面：

hugo new about.md

创建第一篇文章，放到 post 目录，方便之后生成聚合页面。

hugo new post/first.md

安装主题

到 Complete List | Hugo Themes (gohugo.io) 挑选一个心仪的主题，比如你觉得 Jane | Hugo Themes (gohugo.io) 不错，找到相关的 GitHub 地址，创建目录 themes，在 themes 目录里把主题 git clone 下来3：

git clone https://github.com/xianmin/hugo-theme-jane.git --depth=1 themes/jane

运行hugo

hugo server

部署到GitHub上

假设你需要部署在 GitHub Pages 上，首先在GitHub上创建一个Repository，命名为：xdls.github.io （xdls替换为你的github用户名）。不要Add a README file。

当我们开发完自己的站点后，我们需要将其构建打包我静态工程，以方便使用Nginx等Web服务器托管4。

hugo

就是这么简单，直接hugo，然后将public下的内容拿去部署即可。

$ git init
$ git remote add origin https://github.com/xdls/xdls.github.io.git
$ git add -A
$ git commit -m "first commit"
$ git push -u origin master
